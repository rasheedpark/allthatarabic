## 원칙 Tenet

올댓아라빅은 아랍어의 어형(sarf)과 구문(nahw)을 패턴화하고, 학습자가 이를 하나의 덩어리(chunk)로 인식한 뒤 직접 말하는 것을 통해 자연스럽게 체득하도록 설계된 강의다. 인지 부담을 최소화하고, 배운 패턴에 어휘를 바꿔 넣어 스스로 문장을 만들고 말할 수 있게 되는 것이 핵심 목표다.

### 강의 원칙

**패턴 Pattern**
패턴은 학습자가 즉시 인지하고 바로 사용할 수 있어야 한다. 복잡한 문법 분석보다 덩어리(chunk) 단위의 인지를 우선하며, 설명은 패턴을 이해하는 데 필요한 최소한으로만 한다. 학습자는 패턴을 '외우는' 것이 아니라 '말하면서 익히는' 것을 목표로 한다.

**드릴 Drill**
드릴은 말하기를 위한 것이다. 아랍어 문장을 한국어로 해석하거나 번역하지 않는다. 배운 패턴에 어휘를 바꿔 넣어 반복해서 말하고, 이전에 배운 패턴과 결합해 연습함으로써 자연스러운 간격 반복(spaced repetition)이 이루어지도록 한다.

**표현 Expression**
교과서 아랍어가 아닌, 아랍인이 실제로 쓰는 어휘와 표현을 다룬다. 필요한 경우 구어체(ʿāmmiyya)를 포함하며, 현지에서 통용되지 않는 표현은 쓰지 않는다. 학습자는 이 과정에서 언어의 실제 쓰임을 익힌다.

**유추 Inference**
배운 패턴을 스스로 떠올리고 적용할 수 있는 다양한 문제를 제공한다. 답을 알려주기보다 맥락 안에서 스스로 생각하고 말하게 유도하는 것이 목적이며, 이 과정이 실질적인 언어 운용 능력을 만든다.

### 선생님의 역할

선생님은 지금 이 학습자들의 수준을 실시간으로 읽고, 그에 맞는 예문과 드릴을 제공해야 한다. 슬라이드의 순서와 내용은 구조를 잡아주는 틀일 뿐이며, 학습자의 반응에 따라 반복의 깊이와 속도를 조율하는 것은 선생님의 판단이다.

드릴과 유추 문제는 간격 반복의 원리에 따라 이전 패턴을 꾸준히 재등장시키도록 구성되어야 하며, 선생님은 이를 자연스럽게 유도해야 한다.

수업의 긴장을 풀기 위해 내러티브(narratives)를 적절히 활용한다. 아랍어 표현, 아랍인의 문화, 언어적 배경에 관한 짧은 이야기들은 학습자의 흥미와 동기를 유지하는 역할을 하며, 이는 학습자 주도의 본 수업과 분명히 구분되어 진행한다.

## 구조 Structure

올댓아라빅의 수업은 패턴(Pattern) — 드릴(Drill) — 표현(Expression) — 지문(Nass)의 네 가지 요소로 구성된다. 이 순서는 학습자가 새로운 언어 구조를 처음 인지하는 단계부터, 체화하고, 실제 맥락에서 유추하는 단계까지를 하나의 흐름으로 설계한 것이다.

**패턴 Pattern**
해당 유닛의 핵심 구문을 하나의 슬라이드로 제시한다. 아랍어 구문과 한국어 의미, 필요한 경우 짧은 설명(note)이 함께 표시된다. 패턴 슬라이드 다음에는 레퍼토리(Repertory) 슬라이드들이 이어지며, 같은 패턴 구조에 다양한 어휘를 대입한 예문들을 연달아 보여준다. 수업에서는 문법적 설명을 최소로 하고, 선생님이 먼저 발음을 들려준 뒤 학습자가 따라 말하면서 패턴의 형태와 의미를 동시에 받아들이게 한다. 이해보다 인식이 먼저다.

**드릴 Drill**
패턴 슬라이드 다음에 이어지는 연습 슬라이드들이다. 같은 패턴 구조 안에서 어휘를 바꾼 문장들이 제시되며, 이전 유닛에서 배웠던 패턴과 결합된 문장도 포함된다. 한국어 해석은 학습자가 직접 확인하도록 숨겨져 있다. 수업에서는 선생님이 한국어 어휘나 상황을 제시하면 학습자가 아랍어로 즉시 말하는 방식으로 진행한다. 해석하는 것이 아니라 말하는 것이 목표다. 이전에 배운 패턴과 결합해 더 긴 문장을 만드는 연습도 이 단계에서 이루어지며, 이 과정이 자연스러운 간격 반복(spaced repetition)이 된다.

**표현 Expression**
해당 유닛의 모든 서브패턴이 끝난 뒤 등장하는 슬라이드들이다. 패턴 중심이 아닌, 실제 아랍인들이 쓰는 다양한 표현과 어휘를 다룬다. 구어체(ʿāmmiyya)가 포함되기도 하며, 교과서에서는 접하기 어려운 생활 밀착형 표현들로 구성된다. 드릴보다 가볍게, 듣고 따라 말하거나 선생님이 맥락을 설명하면서 진행한다.

**지문 Nass**
표현 다음에 등장하는 듣기·읽기 슬라이드다. 배웠던 패턴들이 결합된 짧은 지문과, 그에 관한 아랍어 질문(sual)이 함께 제시된다. 지문은 처음에 숨겨져 있고 학습자가 질문을 먼저 보고 생각한 뒤 지문을 확인하는 구조다. 학습자는 배웠던 패턴들을 근거로 의미를 스스로 캐치한 뒤 아랍어로 대답한다. 선생님은 답을 바로 알려주지 않고 학습자가 패턴을 활용해 스스로 말할 수 있도록 유도하며, 필요한 경우 어휘 힌트나 유사 예문을 먼저 줄 수 있다.

**유닛 전체 흐름**

데이터시트(datasheet)에 입력된 순서가 곧 슬라이드 순서다. 유닛별로 `status=confirmed`이고 타입이 `pattern / repertory / writing / drill / exp` 중 하나인 행을 시트 순서 그대로 슬라이드로 구성한다. 그 뒤에 해당 유닛의 나스(nass)가 이어진다.

```
[데이터시트 — 시트 순서대로]
pattern, repertory, writing, drill, exp 행
(status=confirmed인 것만, 유닛 기준 필터)

↓

지문 Nass
(나스탭에서 ptn_key 기준 로드)
```

- 슬라이드 순서 재정렬 없음 — 시트에 입력된 순서가 정답
- `writing` 타입: Dongol 폰트(Dongol-Regular.otf) 사용

## 나스 작성 규칙 Nass Writing Rules

### 콘텐츠 범위 (핵심 원칙)

나스에 등장하는 **모든 어휘와 표현은 반드시 스프레드시트에 `status=confirmed`로 확정된 항목만 사용**한다. draft 상태의 항목은 자연스러워 보여도 절대 사용하지 않는다.

| 사용 가능 항목 | 조건 |
|---|---|
| 어휘 | `type=kalimat`, `status=confirmed` — 해당 유닛까지 누적 전체 |
| 표현 | `type=exp`, `status=confirmed` — 해당 유닛까지 누적 전체 |
| 패턴 구조 | `status=confirmed` 패턴 — 해당 유닛까지 누적 전체 |

### 제작 순서 (반드시 준수)

```
1. 칼리마트 확정  →  kalimat status = confirmed
2. 익스프레션 확정  →  exp status = confirmed
3. 나스 작성  →  위 확정 항목만 사용
```

이 순서를 지키지 않으면 나스에 미확정 어휘가 섞여드는 오류가 발생한다.

### 씬 조건

1. **실제 씬 (핵심)**: 실제 아랍인들이 나눌 법한 대화여야 한다. 교재용 인공 예문 금지.
2. **새 패턴 노출**: 해당 유닛에서 새로 배운 패턴이 씬 안에 자연스럽게 등장해야 한다.
3. **새 콘텐츠 중심**: 이번 유닛의 신규 kalimat·expression이 씬의 중심이 되어야 한다.
4. **이전 콘텐츠 재활용 가능**: 이전 유닛 내용은 `status=confirmed`인 kalimat·expression 범위 안에서만 포함할 수 있다.
5. **카테고리**: 나스의 성격에 따라 유형 분류. 유형은 추후 확장.

| 유형 | 설명 |
|---|---|
| 듣고 따라하기 | 지문을 듣고 발화를 따라하는 형식 |
| 질문-대답 (sual-jawab) | 질문을 먼저 보고 지문에서 답을 찾는 형식 |
| 마지막 말 고르기 | 대화 흐름을 읽고 마지막 발화를 선택하는 형식 |
| (추후 추가) | — |

## 데이터 모델 Data Model

올댓아라빅의 콘텐츠는 구글 스프레드시트에 입력되며, 앱은 이 데이터를 읽어 슬라이드를 자동으로 생성한다. 스프레드시트는 **세 개의 탭**으로 구성된다.

| 탭명 | 역할 |
|---|---|
| `pattern` | 패턴 유닛 정의. 레퍼런스 시트 |
| `datasheet` | 모든 슬라이드 콘텐츠 통합 (패턴·레퍼토리·드릴·표현·지문·어휘) |
| `나스탭 (Nass Tab)` | 나스 퀴즈 데이터 별도 관리 (id, sual, alif, ba, jim, dal, answer) |

`datasheet`와 `나스탭`은 **완전히 분리된 탭**이다. 나스 슬라이드 본문은 datasheet에, 퀴즈 문항은 나스탭에 입력한다. 앱은 나스 id를 기준으로 두 탭의 데이터를 결합한다.

모든 탭에서 `U` 값을 공유한다. `U`는 패턴 식별자이자 유닛 번호를 겸하며, 사용자가 직접 입력한다.

### 패턴 시트 Pattern Sheet (탭명: `pattern`)

패턴 유닛을 정의하는 레퍼런스 시트다. 데이터시트의 `ptn_key` 컬럼은 이 시트를 참조해 자동으로 채워진다.

| 컬럼 | 설명 |
|---|---|
| `C` | 챕터 번호 |
| `U` | 패턴 식별자. 사용자가 직접 입력. 유닛 번호와 패턴 키를 겸함 (예: `P00`, `P01`, `G00`, `A01`) |
| `arabic` | 아랍어 패턴 |
| `korean` | 한국어 의미 또는 로마자 발음 표기 |
| `note` | 보조 설명 |
| `type` | 패턴 계열 지정. 괄호 안 알파벳이 계열 코드 (예: `pattern(p)`, `pattern(a)`, `pattern(g)`) |
| `ptn_key` | 로마자 발음 표기. 데이터시트 ptn_key 자동 조회의 참조값 (예: `bā tā thā nūn yā`) |

**U값 계열별 번호 시작 규칙**

| 계열 | 예시 | 시작 번호 |
|---|---|---|
| `P` (pattern(p)) | `P00`, `P01`, `P02` ... | **00**부터 |
| `G` (pattern(g)) | `G00`, `G01` ... | **00**부터 |
| `A`, `B`, `C` ... | `A01`, `A01a`, `B01` ... | **01**부터 |

서브패턴은 `A01`, `A01a`, `A01b` 처럼 알파벳 접미사로 구분한다.

### 데이터시트 Datasheet (탭명: `datasheet`)

모든 슬라이드 콘텐츠를 하나의 시트에 통합한다. 패턴 슬라이드(`type = pattern(x)`)도 이 시트에 함께 입력한다.

#### 컬럼 구조

| 컬럼 | 설명 |
|---|---|
| `C` | 챕터 번호 |
| `U` | 패턴 식별자. 패턴 시트와 동일한 값을 사용자가 직접 입력 (예: `P01`, `A01a`) |
| `arabic` | 아랍어 텍스트. 슬라이드 메인 영역에 표시 |
| `korean` | 한국어 의미 또는 설명 |
| `note` | 보조 설명. 슬라이드 하단 작은 글씨로 표시 |
| `type` | 슬라이드 유형. 드롭다운으로 선택 (아래 표 참조) |
| `status` | 확정 여부. `confirmed`인 항목만 슬라이드에 포함 |
| `id` | 자동 생성 ID. 형식: `{U}-{타입축약}{순번}` (예: `P01-pat01`, `P01-rep01`) |
| `ptn_key` | 자동 조회. 패턴 시트에서 U값 기준으로 불러오는 로마자 설명 |
| `url` | 오디오 파일 URL. 비어있으면 GCS 자동 경로 사용 |
| `lahja` | 방언 태그. `MSA` / `LEV` / `EGY` 등 |
| `중복` | 중복 여부. 체크 시 슬라이드 제외 |
| `script` | 수업 스크립트. 선생님 전용, 슬라이드 미표시 |

#### type 값과 id 축약어

| type | 설명 | id 축약어 | id 예시 | ptn_key 사용 |
|---|---|---|---|---|
| `pattern(p)`, `pattern(a)` 등 | 패턴 슬라이드 | `pat` | `P01-pat01` | — |
| `repertorie` | 레퍼토리 슬라이드 | `rep` | `P01-rep01` | ✓ |
| `drill` | 드릴 슬라이드 | `drl` | `P01-drl01` | ✓ |
| `exp` | 표현 슬라이드 | `exp` | `A01-exp01` | — |
| `nass` | 지문 슬라이드 | `nas` | `A01-nas01` | — |
| `kalimat` | 어휘 (슬라이드 미노출) | `kal` | `A01-kal01` | — |

#### id 자동 생성 수식 (datasheet H열)

```
=IFERROR(IF(OR(B2="",F2=""),"",B2&"-"&IF(LEFT(F2,7)="pattern","pat",IFS(F2="repertorie","rep",F2="drill","drl",F2="exp","exp",F2="nass","nas",F2="kalimat","kal",TRUE,LEFT(F2,3)))&TEXT(COUNTIFS($B$2:B2,B2,$F$2:F2,F2),"00")),"")
```

#### ptn_key 자동 조회 수식 (datasheet I열)

```
=IFERROR(INDEX(pattern!$G:$G,MATCH(B2,pattern!$B:$B,0)),"")
```

### 칼리마트 Kalimat

데이터시트 안에서 `type = kalimat`으로 관리되는 어휘 데이터다. 슬라이드에는 노출되지 않으며, 핸드아웃과 단어장 자동 생성에 활용된다.

**어휘 선정 방식은 두 가지다.**
- **패턴 기준** — 해당 패턴을 설명하기 좋은 어휘를 미리 선정한다
- **콘텐츠 기준** — 표현(exp)과 지문(nass)까지 보고, 자연스럽게 연결되는 어휘를 추가한다

두 방향 모두 진행해야 한 유닛의 어휘가 완성된다. 알파벳 단계 이후 유닛당 약 20개를 유지하며, 이미 등장한 어휘가 다시 입력되지 않도록 `중복` 컬럼으로 체크한다.

---

### 연결 규칙

**레퍼토리 / 드릴 → 패턴**
`U` 값이 일치하는 패턴 슬라이드에 묶인다. 슬라이드 순서는 `id` 컬럼 순서를 따른다.

**표현 / 지문 / 어휘 → 유닛**
`C` + `U`(유닛 단위 값) 조합으로 해당 유닛에 속하는 항목들을 필터링한다.

**슬라이드 생성 순서**
```
유닛 내 서브패턴 (U값 알파벳순: A01 < A01a < A01b)
├── 패턴 슬라이드 (type=pattern(x), U값 일치)
├── 레퍼토리 슬라이드 (type=repertorie, U값 일치, id순)
└── 드릴 슬라이드 (type=drill, U값 일치, id순)

(서브패턴 반복)

표현 (type=exp, 유닛 기준, id순)
지문 (type=nass, 유닛 기준, id순)
```

**오디오 URL 우선순위**
`url` 컬럼에 값이 있으면 해당 URL 사용. 비어있으면 GCS 자동 경로 사용:
`https://storage.googleapis.com/all-that-arabic-14/{type}/{id}.mp3`

**텍스트 마크업**

| 문법 | 표시 방식 |
|---|---|
| `_텍스트_` | 청록색 테두리 박스 |
| `*텍스트*` | 청록색 굵은 글자 |
| `\n` | 줄바꿈 |

## 콘텐츠 확장 Content Expansion

올댓아라빅의 수업 데이터는 강의 이외의 콘텐츠 포맷으로 파생된다. 같은 데이터를 기반으로 하되 목적과 포맷이 다른 두 가지 콘텐츠를 계획한다.

**다르쓰 Darss — 인터넷 강의**
패턴과 레퍼토리의 확장이다. 수업에서 다룬 핵심 패턴과 예문을 기반으로 제작하는 녹화 강의로, 슬라이드에 선생님이 등장할 공간을 별도로 확보한 레이아웃으로 재구성한다. 오프라인 수업을 보완하거나 대체하는 형태로 활용할 수 있으며, 챕터별 링크 공유가 가능하다. 슬라이드 → 강의용 이미지 파일 → 영상 편집의 순서로 제작 프로세스가 이어진다.

**칼람 Kalam — SNS / 커뮤니티 콘텐츠**
표현과 지문의 확장이다. 함께 배우면 좋은 표현들을 묶어 짧고 재미있는 콘텐츠로 재가공한다. 어렵거나 무겁지 않게, 실제 아랍인들의 언어와 문화를 접하는 가벼운 포맷이 목적이다. 마스터 시트의 expression 데이터를 그대로 활용할 수 있어 별도 콘텐츠 제작 부담이 낮다.

## 제품 확장 Product Expansion

올댓아라빅의 구조와 데이터 모델을 중심으로 인접한 학습 수요를 다루는 새로운 제품군으로 수평 확장하고, 현재 웹 기반 앱을 고도화하는 수직 확장을 병행한다.

**수평 확장 — 인접 제품군**
올댓아라빅을 중심으로 문법 과정, 어휘 과정, 올댓아라빅 심화버전 또는 워크북 등 다양한 수평적 제품군으로의 확장을 염두에 두고 설계한다. 각 제품은 마스터 시트의 데이터를 공유하거나 파생하여 활용할 수 있다.

**수직 확장 — 앱 고도화**
현재의 웹 기반 슬라이드 앱은 수업 진행 도구로서의 역할을 한다. 장기적으로는 링글, 스픽과 같은 네이티브 앱 형태로 발전시켜 학습자가 독립적으로 사용할 수 있는 플랫폼으로 수직 확장한다. 마스터 시트 기반의 모듈화된 데이터 구조는 이 전환의 핵심 자산이 된다.

## 다르쓰 제작 프로세스 Darss Production Process

다르쓰는 수업용 슬라이드를 기반으로 제작하는 인터넷 강의 콘텐츠다. 자동화된 슬라이드를 그대로 쓰는 것이 아니라, 선생님이 화면에 등장할 수 있도록 레이아웃을 재구성한 이미지 파일로 변환하는 과정을 거친다.

---

### 제작 단계

```
1. 수업용 슬라이드 완성 (앱 자동 생성)
        ↓
2. 다르쓰 레이아웃으로 변환
        ↓
3. 이미지 파일 내보내기
        ↓
4. 영상 편집 완성
```

---

### 레이아웃 변환 규칙

수업용 슬라이드 레이아웃을 기준으로 다음과 같이 조정한다.

**유지하는 요소**
- 상단 코너 레이블 (유닛명, 슬라이드 타입) — 위치 그대로
- 하단 코너 레이블 (저작권, 로고) — 위치 그대로
- 진행 바

**변경하는 요소**
- 슬라이드 좌측에 전체 너비의 약 **10%** 공간을 비워 선생님 영상 합성 공간으로 확보
- 기존 메인 콘텐츠 영역 전체가 오른쪽으로 이동하여 나머지 **90%** 공간 안에 배치
- 콘텐츠의 크기·비율은 그대로 유지, 위치만 오른쪽으로 shift

```
┌──────────────────────────────────────────┐
│[유닛명]                        [PATTERN] │  ← 코너 레이블 유지
│                                          │
│        │                                 │
│  선생님 │     아랍어 텍스트 (메인)        │
│  영상  │     한국어 의미                  │
│  공간  │     노트                         │
│ (10%) │          (90%)                  │
│                                          │
│© Markaz Arabic                  [로고]  │  ← 코너 레이블 유지
└──────────────────────────────────────────┘
```

### 내보내기 형식

- 파일 형식: PNG (투명 배경 불필요, 배경색 포함)
- 해상도: 1920×1080 기준 (16:9)
- 파일명: `{id_ptn}-{type}-{순번}.png` 형식으로 자동 생성
- 저장 위치: GCS 버킷 `/darss/{챕터}/{유닛}/`

### 영상 편집 흐름

이미지 파일이 준비되면 영상 편집 단계에서 선생님 영상을 좌측 10% 공간에 합성한다. 슬라이드 전환, 오디오 믹싱, 자막 삽입은 편집 단계에서 처리한다.
